Step 4: Sequence

When you want more than one effect to take place in an animation, you need to coordinate timing and duration. JSTween does this with the appropriately named time and duration attributes.

The time attribute specifies at how many seconds into the sequence you want the effect to begin. For example, we have been using time: 0 in the previous examples, and all the animations start as soon as you click the "Test Code" button. However, if you wanted an animation to delay or happen later in the sequence, you simply specify a later time such as time: 3.

Delay Start:

Test Code
  • $('#luke').tween({
    • left: {
      • start: 0,
      • stop: 175,
      • time: 3,
      • units: 'px',
      • duration: 1,
      • effect: 'easeInOut'
    • }
  • });
  • $.play();

In this case, when you click "Test Code", nothing happens for three seconds. On its own, this wouldn't be the best feature to use, since users may think the website isn't working properly. However, if you want to sequence actions so that one animation can happen after another, it is a useful tool.

The duration attributes specifies how long a particular animation will take. Until now, we have been using duration: 1, and all our examples have taken one second to complete. However, to make your animations feel more lifelike, it is often useful to change their duration. For example, specifying duration: 3 makes Luke take a lot longer to move.

Slow Move:

Test Code
  • $('#luke').tween({
    • left: {
      • start: 0,
      • stop: 175,
      • time: 0,
      • units: 'px',
      • duration: 3,
      • effect: 'easeInOut'
    • }
  • });
  • $.play();

Animations can be piled on top of each other by specifying the same time for each one. For example, if I wanted Luke to jump through the air while spinning, I would combine move and rotate as shown below.

Jump and Spin:

Test Code
  • $('#luke').tween({
    • left: {
      • start: 0,
      • stop: 200,
      • time: 0,
      • units: 'px',
      • duration: 1,
      • effect: 'easeInOut'
    • },
    • rotate: {
      • start: 0,
      • stop: 360,
      • units: 'degrees'
      • time: 0,
      • duration: 1,
      • effect: 'easeInOut'
    • }
  • });
  • $.play();

The rotation is performed around the middle of the object, which is why it appears like Luke is jumping even though we only told him to move horizontally using left. Notice that each directive is separated by a comma. You can chain as many of these together as you like. Since we wanted Luke to both move and spin at the same time, we kept both time attributes set at 0. However, this does not always have to be the case.

If I want one effect to happen after another, I simply specify two different times for the animations to occur. The following example has two shadow effects happen within one second of each other.

Glow and Fade:

Test Code
  • $('#luke').tween({
    • shadow: {
      • start: '0px 0px 0px #000',
      • stop: '10px 10px 30px #00f',
      • time: 0,
      • duration: 1,
      • effect: 'easeInOut'
    • },
    • shadow: {
      • start: '10px 10px 30px #00f',
      • stop: '0px 0px 0px #000',
      • time: 1,
      • duration: 1,
      • effect: 'easeInOut'
    • }
  • });
  • $.play();

The shadow attribute can be used to both fade in and fade out. In this case, we accomplished both by fading the shadow in at time: 0, and fading the shadow out at time: 1. Time and duration are measured in seconds, and decimal values such as 0.5 are accepted as well.

Timing is one of the most critical parts of animation, so it may take a lot of experimentation before you get a sequence that feels "natural". I noticed that making my shadow effects faster than my motion effects (duration: 0.5 and duration: 1, respectively) resulted in more realistic animation. This helpful webinar from Pixar Animation Studios can help you learn the basics of timing and spacing.

Once you think you have a good grasp of how to chain animations together, move on to Step 5.
